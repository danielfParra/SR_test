Decoding the Message

In the experiment, there will be two types of Player B, which we will call Decoder and Direct. The Decoder will have to decode the message sent by Player A. The Direct will receive the message directly.

Both types will receive the following type of message from Player A:

Player A's message:

To receive the highest payoff, you should guess X.

The Direct type will directly observe the number that Player A intended to send. The Decoder will instead see the message in the form of a mathematical expression. The decoding tasks will be of similar difficulty as the ones you solved at the beginning of the experiment. For example, if Player A intended to send the number 3, the Decoder might see the message To receive the highest payoff, you should guess 2×3 − 3.

The Decoder then needs to answer two questions:

  1. What is the number that Player A intended to send?
  2. What is the secret number?
Importantly, the Decoder cannot submit a guess about the secret number without first correctly answering the initial question.

Please note the following:
  • Player B has {{ Constants.TIME_PER_ROUND }} seconds to submit a guess. If a Decoder does not manage to do this because they spend too much time decoding the message, Player A's number will be submitted as their guess.
  • A Decoder can only submit an answer to the mathematical expression every {{ Constants.ATTEMPT_DELAY }} seconds. Therefore, it is typically not a good idea to try to solve the mathematical expression by simply guessing it.
